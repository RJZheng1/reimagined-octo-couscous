Brief History of UNIX

Back when computers were large, software was always customized to serve a specifc purpose, and software for one given system didn't run
on another system, making it difficult for both the users and the administrators.

In 1969 a team of developers in the Bell Labs laboratories started working on a solution problem to address these issues.

(1)Simple and Elegant (2) Written in C (3) Able to recycle code

Types of Shell in UNIX

sh

 Aka the "Bourne-Shell" one of the original shells developed by Stephen Bourne at AT&T's Bell Labs in 1977
Some features include: input and output redirection, shell scripting with string and integer variables and condition testing and looping

bash

 Enhancements on the sh, aka the "Bourne-again Shell" based on sh
Can run sh shell scripts unchanged. Command completion and command history (new additions from sh)

csh and tsch

 csh developed by Bill Joy
Ken Greer developed tsch in 1978, which fixed problems in csh and added command completion (guess as you type).
tsch does not run bash scripts

ksh

 Developed by David Korn, about the time tsch was introduced.
Improves on Bourne shell by adding floating point arithmetic, job control command aliasing and command completion.

Let's Get Started

Every bash script begins with #!/bin/bash.
This is known as a shabang and notifies the Unix shell what program should be run to interpret the script.

Let's start with a simple Hello World script.
#!/bin/bash
echo "Hello World"            #prints "Hello World\n"
Bash has two methods for printing things: echo and printf.
Echo simply prints out the argument, adding a new line character.
Printf, on the other hand, allows for definition of a formatting string, much like the printf in C.
Comments begin with a #.

Variables in Shell

To process data/information, data must be kept in the computer's RAM. RAM is divided into small locations,
and each location had unique number called an address, which is used to hold our data. Programmers
can give a unique name to this address called a variable.

Creating Variables

Creating a variable is simple. Just do VARNAME="value".
Variable names are case-sensitive and can contain digits, but the first character cannot be a digit.
One thing to note is that you cannot put spaces around the equal sign.
This is due to the way that bash interprets scripts. It breaks each line up into tokens separated by spaces and then interprets them.
The variable value does not have to be surrounded by quotes, but it is a good idea to do so.
There are four types of variables: integers, arrays, strings, and constants.
You can use declare -a VARIABLE to make a variable an array, -i for integers, and -r for constants.

Arrays

There are a few ways to declare an array.
ARRAY[INDEX]=VALUE
declare -a ARRAY
ARRAY=(value1 value2 ...)
To get the values of an array, you must use the following syntax: $ARRAY[INDEX]. * can be used as an index to get all the members.
You can use unset to remove elements from an array. unset ARRAY[INDEX]. If no index is specified, the entire array is removed.

Shell Expansion

Single quotes are used to preserve the literal value of the characters inside. A single quote cannot be enclosed between them, even if backslash escaped.
Double quotes also preserve the values, except for the dollar sign, back ticks, and backslash, which have special meanings.
The backslash is used for escape sequences, listed here.
The dollar sign is used to get the value of a variable enclosed within: ${VAR}
The backtick is used to return the output of a command: `date` will return the date.
This can also be done with the dollar sign: $(date)
Arithmetic operations can be done on using the following syntax: $[EXPRESSION]. List of operators

Conditionals

If Statements

The general syntax for an if statement is
if CONDITION; then
        COMMANDS
    elif CONDITION; then
        COMMANDS
    else
        COMMANDS
fi
A list of conditions that can be checked can be found here.
Conditions can be simply a command. If so, then the veracity will be based on the exit status. If the exit status is 0 (presumably executed correctly), then it's true, otherwise false.
If the condition is an arithmetic value (enclosed in double parantheses), 0 is false and other numbers are true.
As usual, ! negates the value. -a is an and statement and -o is an or statement.
The if-else structure can be shortened to CONDITION && TRUE_COMMANDS || FALSE_COMMANDS

Case Statements

A case statement is used to compare a single expression to multiple values. Its syntax is as follows 
case EXPRESSION in 
CASE1)
    COMMANDS
    ;;
CASE2)
    COMMANDS
    ;;
esac

Loops

For Loops

The for loop is one of the three shell looping constructs. This loop allows for specification of a list of values.
A list of commands is executed for each value in the list. The general syntax for a for loop is as follows
for name in LIST; do 
        COMMANDS;
done

While Loop

The while loop executes until the test-command fails. Syntax is as follows
while CONTROL-COMMAND; do
        CONSEQUENT-COMMANDS;
done

Until Loop

The until loop is very similar to the while loop, except that the loop executes until the test-command executes successafully.
It can pretty much be thought of as the opposite of the while loop. As long as the condition is false (keeps on failing), the loop will continue.
The general syntax for an Until loop is as follows
until TEST-COMMAND; do
        CONSQUENT-COMMANDS;
done

Writing a Function

Shell functions are a way to group commands for later execution, using a single name for this group, or routine.
The name of the routine must be unique within the shell or script. These functions are executed like regular commands.
Two general syntax for creating a function is as follows
function FUNCTION_NAME {
    COMMANDS;
}
OR 
FUNCTION_NAME () { 
    COMMANDS;
}
In order to get parameters, within the function body, you must use ${PARAMETER}, where parameter is the position of the parameter, starting with 1 for the first parameter.
